# 临床医技 360 核心版 — 设计规格

> **项目名称：** 医疗机构医技数据中台与患者360平台
> **阶段：** 一期 — 临床医技 360 核心版
> **设计日期：** 2026-04-24

---

## 1. 项目宗旨

让临床医生和医技科室医生能够在一个统一平台上，可靠地查看患者的检验与影像信息，并获基础的 AI 辅助阅读能力，同时确保数据隔离与权限合规。

一期之后，二期扩展 AI 辅助与审计增强，三期扩展 BI 分析与运营驾驶舱。

---

## 2. 一期范围

### 2.1 必做

- 统一中台接入层（支持 LIS / RIS / PACS / EMR 接入）
- 检验数据采集与标准化
- 影像检查数据采集与标准化
- 患者主索引关联（统一患者标识）
- 患者360汇总页
- 患者时间轴页（检验 + 影像混合排列）
- 检验详情页（指标明细、参考区间、异常标记、趋势对比）
- 影像报告详情页（报告内容、结论、PACS 跳转入口）
- 轻量 AI 辅助摘要（检验异常摘要、影像报告摘要、患者近期摘要）
- RBAC 功能权限
- 科室/岗位数据权限
- 基础操作审计日志

### 2.2 明确排除

- 全医技全量接入（只覆盖检验 + 影像）
- 重型 BI 平台
- 自由问答型医疗大模型入口
- 诊断决策支持或自动诊断建议
- 完整审批工作流
- PACS 原生阅片替代能力

## 4. 系统架构

```
┌─────────────────────────────────────────────────┐
│              临床应用前台 (React Frontend)         │
│  患者360汇总页 | 时间轴 | 检验详情 | 影像详情     │
├─────────────────────────────────────────────────┤
│              权限控制层 (横切三层)                │
│  RBAC 功能权限 | 科室/岗位数据权限                │
├────────────────────┬────────────────────────────┤
│   统一中台接入层    │      AI 辅助服务            │
│   - 接入适配器     │      - 检验异常摘要          │
│   - 标准化映射     │      - 影像报告摘要          │
│   - 主索引关联     │      - 患者近期摘要          │
│   - 统一查询接口   │                              │
├────────────────────┴────────────────────────────┤
│              数据存储与索引层                     │
└─────────────────────────────────────────────────┘
```

### 3.1 架构原则

- **前台不隔针对各业务系统。** 临床应用前台只通过统一中台查询数据，不直接对接 LIS/RIS/PACS/EMR。
- **AI 基于标准化数据工作。** AI 辅助服务从统一中台获取已标准化的数据，不直接读取各系统原始数据。
- **权限横切。** 功能权限和数据权限统一控制，不在前端分散实现权限判断。
- **每个模块有明确边界。** 接入层、业务层、AI 层、权限层职责不重叠。

## 5. 核心模块

### 5.1 系统接入模块

- 支持多数据源适配器模式
- 基础适配器：LIS 适配器（检验数据）、RIS/PACS 适配器（影像检查数据）、EMR 适配器（患者/就诊信息）
- 字段映射表：将各系统字段映射到统一标准字段
- 增量同步机制：支持按时间戳轮询或事件通知进行增量同步
- 失败重试：自动重试 + 人工干预队列

### 5.2 主数据与患者索引模块

- 统一患者标识（基于身份证号、就诊卡号、姓名+生日等多条件匹配）
- 就诊关联（一个患者关联多次就诊/住院/门诊）
- 患者维度信息包括：姓名、性别、年龄、身份证号、联系方式等

### 5.3 患者360浏览模块

- 患者基础信息区
- 关键提醒区（近期异常检验、重要影像检查、待关注结果）
- 检验摘要区（最近检验结果、异常项数量、趋势入口）
- 影像摘要区（最近影像检查、报告结论、查看详情入口）
- AI 摘要区（患者近期医技检查摘要）

### 5.4 患者时间轴模块

- 按时间倒序展示检验与影像事件
- 每个事件标签：类型、科室、状态、结果摘要
- 筛选：时间范围、检验/影像、科室、异常/全部

### 5.5 检验详情模块

- 检验单基本信息
- 指标明细列表（含参考范围、异常高亮）
- 历史趋势对比（最近多次检验结果对比）

### 5.6 影像详情模块

- 检查基本信息
- 报告内容与结论
- AI 报告摘要
- 影像系统（PACS）跳转入口

### 5.7 权限控制模块

- 功能权限：控制页面和按钮访问
- 数据权限（科室/岗位）：控制可查看的患者范围
  - 临床医生：按本科室在管患者查看
  - 医技科室医生：按申请来源科室或本科室业务数据查看
  - 平台管理员：全权限
- 权限判断由后端统一裁决，前端仅根据接口返回做展示

### 5.8 轻量 AI 辅助模块

- 检验异常摘要（异常项汇总，而非自由解释）
- 影像报告摘要（浓缩报告文本）
- 患者近期医技结果概览
- 所有 AI 输出显式标记”辅助生成，仅供参考”
- AI 调用可追溯：记录生成时间、输入来源、模型版本

## 6. 用户角色

| 角色         | 职责               | 页面权限                                    | 数据权限           |
| ------------ | ------------------ | ------------------------------------------- | ------------------ |
| 临床医生     | 查看患者360        | 患者360、时间轴、检验详情、影像详情         | 本科室在管患者     |
| 医技科室医生 | 查看检验/影像结果  | 患者360（有限）、时间轴、检验详情、影像详情 | 本科室业务相关患者 |
| 科室管理员   | 管理科室用户与权限 | 页面 + 本科室权限管理                       | 本科室数据         |
| 平台管理员   | 接入配置、全局权限 | 系统管理全量                                | 全部数据           |

## 7. 核心数据模型

### 7.1 Patient

- id, unified_patient_id
- name, gender, birth_date, id_card_no
- phone, address

### 7.2 Encounter

- id, patient_id
- encounter_type (门诊/住院/急诊)
- department_id, visit_datetime
- admission_reason

### 7.3 LabOrder / LabResult

- LabOrder: id, patient_id, encounter_id, order_no, department_id, order_datetime, status, specimen_type
- LabResult: id, lab_order_id, item_code, item_name, result_value, unit, ref_range_low, ref_range_high, abnormal_flag, result_datetime
- 历史趋势可基于 item_code 跨 LabOrder 查询

### 7.4 ImagingOrder / ImagingReport

- ImagingOrder: id, patient_id, encounter_id, order_no, department_id, body_part, modality, order_datetime, status
- ImagingReport: id, imaging_order_id, report_content, impression, report_doctor, report_datetime, pacs_url

### 7.5 ClinicalEvent

- 时间轴统一事件模型
- id, patient_id, event_type (lab/imaging)
- event_datetime, event_summary, event_status
- reference_id (关联对应业务 ID)
- department_id

### 7.6 User / Role / Department / Position

- User: id, username, name, department_id, position_id
- Role: id, name, permissions (JSON 或关联表)
- Department: id, name, parent_id
- Position: id, name, department_id, data_scope

### 7.7 AccessLog

- id, user_id, patient_id, action, request_detail, ip_address, access_datetime

## 8. 一期验收标准

### 临床可用性

- 临床医生可按患者查看检验与影像汇总信息
- 可通过时间轴回看患者检查检验事件序列
- 可进入检验/影像详情查看原始结果与报告
- 可直接从影像详情跳转到 PACS 系统

### 数据可用性

- 检验与影像数据能通过统一中台稳定接入
- 患者身份能跨系统正确关联
- 数据延迟和缺失率在可接受范围内

### 权限可控性

- 不同角色看到的页面、功能和患者范围不同
- 科室/岗位数据权限生效
- 未授权用户不能跨科室查看非授权患者

### AI 可控性

- 可生成轻量摘要
- AI 结果有明确辅助标记
- 原始数据始终完整可见
- AI 不越权输出医学诊断结论
